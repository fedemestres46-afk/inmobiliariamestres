# Checklist De Entrega Comercial

Esta lista sirve para mostrar, vender o entregar una demo funcional sin olvidarte nada importante.

## Antes de mostrar la demo

- Confirmar que la web publica abre bien
- Confirmar que `/admin/login` funciona
- Confirmar que el usuario `owner` puede editar
- Confirmar que una propiedad publicada se ve en:
  - home
  - listado `/propiedades`
  - ficha individual
- Confirmar que el mapa carga propiedades
- Confirmar que una consulta genera un lead
- Confirmar que el historial de actividad muestra cambios recientes

## Datos de demo recomendados

- 4 a 6 propiedades bien cargadas
- 2 o 3 leads de ejemplo
- al menos 1 propiedad con coordenadas y mapa
- al menos 1 propiedad con varios servicios y adicionales
- al menos 1 propiedad con varias imagenes

## Accesos que deberias tener listos

- URL publica
- URL de admin
- usuario y contraseña de demo
- acceso a GitHub
- acceso a Vercel
- acceso a Supabase
- acceso al dominio si corresponde

## Que explicar en una demo

1. Carga de propiedades
2. Edicion y estados
3. Subida de imagenes
4. Ficha publica por propiedad
5. Ingreso de leads
6. Seguimiento desde el mini CRM
7. Historial de cambios
8. Roles de acceso al panel

## Si lo vas a vender

- definir precio de implementacion
- definir si el dominio lo paga el cliente o vos
- definir si Vercel y Supabase quedan a tu nombre o al del cliente
- definir mantenimiento mensual
- definir que incluye soporte y que no

## Entrega minima prolija

- documento con accesos
- breve video o explicacion de uso
- checklist de servicios conectados
- aclaracion de costos mensuales
- plan de soporte post-entrega

## Costos a aclarar

- dominio
- Vercel
- Supabase
- mantenimiento
- cambios evolutivos

## Riesgos que conviene aclarar

- si se cambia la configuracion de Supabase o Vercel sin control, el sistema puede dejar de guardar
- las claves privadas no deben circular por chat
- si se duplica el proyecto para otro cliente, conviene usar otro Supabase y otro Vercel
